Investor Portal — Five Implementations

Five candidate stacks for a gated investor portal, each implemented end-to-end as a separate sub-project under portals/ in this repository. They share the same look, the same minimum feature set (sign-up, email verification, login, protected dashboard, profile + stub deal pipeline) so the only thing that differs is the auth + storage architecture.

What each PoC demonstrates

Each subdirectory ships its own README with the exact create-account + deploy steps. URLs are filled in here as each one goes live.

1. Supabase

vanilla HTML · Supabase JS · GH Pages

Everything in one provider: Postgres + auth + RLS + email verification + storage. Front-end runs on GH Pages and calls Supabase directly.

Live URL — pending deploy

2. Cloudflare Access + D1

CF Pages · CF Access · D1 (SQLite)

Edge-level auth: Cloudflare Access gates the subdomain (email OTP / SSO / MFA). A Pages Function reads the signed Access JWT and persists profile state in D1.

Live URL — pending deploy

code ready

3. Clerk + Neon

CF Pages · Clerk SDK · Neon (Postgres)

Best-in-class hosted sign-up UI from Clerk; profile + deal data live in Neon Postgres, queried via a Pages Function that verifies Clerk JWTs against the JWKS.

Live URL — pending deploy

code ready

4. Vercel + Auth.js + Neon

Next.js · Auth.js v5 · argon2id · Neon

Self-hosted credentials auth. Passwords hashed with argon2id, sessions in HttpOnly cookies, all logic + user table owned by us. Most code, most control.

Live URL — pending deploy

code ready

5. Firebase

vanilla HTML · Firebase Web SDK · GH Pages

Firebase Auth (email/password + verification + MFA) and Firestore for profile + deal docs, locked down by security rules on request.auth.uid. Front-end on GH Pages.
